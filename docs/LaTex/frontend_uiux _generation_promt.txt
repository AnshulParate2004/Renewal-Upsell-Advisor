================================================================================
FRONTEND UI/UX GENERATION PROMPT: S-007 AUTONOMOUS REVENUE AGENT DASHBOARD
================================================================================

PROJECT IDENTITY
----------------
Application Name: S-007 Revenue Command Center
Target Users: Customer Success Managers (CSMs), Revenue Operations Teams
Platform: Web Application (Desktop-first, Responsive)
Design Philosophy: Data-Dense, Action-Oriented, Autonomous Intelligence

DESIGN MISSION
--------------
Create a modern, premium dashboard that empowers CSMs to monitor 150+ accounts effortlessly by surfacing AI-driven insights and autonomous actions. The UI should feel like a "mission control center" for revenue operations - combining real-time intelligence with one-click interventions.

================================================================================

-----------------

- Brand Primary: #2563EB (Electric Blue) - Buttons, CTAs, Active states

1.2 TYPOGRAPHY
--------------
Font Family: 
- Primary: 'Inter', sans-serif (Google Fonts)
- Monospace: 'JetBrains Mono', monospace (for metrics, numbers)

Font Sizes:
- Headline 1: 32px / 2rem (Page titles)
- Headline 2: 24px / 1.5rem (Section headers)
- Headline 3: 20px / 1.25rem (Card titles)
- Body Large: 16px / 1rem (Primary text)
- Body Regular: 14px / 0.875rem (Secondary text)
- Body Small: 12px / 0.75rem (Captions, labels)
- Metric Display: 48px / 3rem (Large numbers, scores)

Font Weights:
- Regular: 400 (Body text)
- Medium: 500 (Subheadings)
- Semibold: 600 (Buttons, labels)
- Bold: 700 (Headlines, emphasis)

1.3 SPACING SYSTEM
------------------
Base Unit: 4px
Scale: 4px, 8px, 12px, 16px, 24px, 32px, 48px, 64px

Padding:
- Card Padding: 24px
- Button Padding: 12px 24px
- Input Padding: 12px 16px
- Section Spacing: 32px

1.4 COMPONENT LIBRARY
---------------------
Buttons:
- Primary: Solid background (#2563EB), white text, rounded-lg (8px), hover lift effect
- Secondary: Outlined border, transparent background, hover fill
- Danger: Solid red background (#EF4444), white text
- Ghost: No border, transparent, hover background

Cards:
- Background: Slate 800 (dark mode) or White (light mode)
- Border: 1px solid Slate 700
- Border Radius: 12px
- Shadow: 0 4px 6px rgba(0, 0, 0, 0.1)
- Hover: Subtle lift (translateY(-2px)) + shadow increase

Badges:
- Pill-shaped (rounded-full)
- Small padding: 4px 12px
- Font size: 12px
- Color-coded by status (Green/Yellow/Red)

Icons:
- Library: Heroicons or Lucide React
- Size: 20px (default), 24px (large), 16px (small)
- Stroke width: 2px

1.5 ANIMATION & MICRO-INTERACTIONS
----------------------------------
Transitions:
- Default: 200ms ease-in-out
- Hover effects: 150ms ease-out
- Page transitions: 300ms ease-in-out

Micro-interactions:
- Button click: Scale down (0.98) on active
- Card hover: Lift effect (translateY(-2px))
- Loading states: Skeleton screens with shimmer effect
- Success actions: Checkmark animation + green flash
- Error states: Shake animation + red border pulse

================================================================================
DESIGN PRINCIPLES SUMMARY
================================================================================

1. **DATA DENSITY**: Show maximum relevant information without overwhelming
2. **ACTION-ORIENTED**: Every insight should have a clear next action
3. **AUTONOMOUS INTELLIGENCE**: Highlight AI-driven insights prominently
4. **REAL-TIME UPDATES**: Use WebSockets for live data (risk scores, calls)
5. **PROGRESSIVE DISCLOSURE**: Show summary first, details on demand (flyouts)
6. **CONSISTENT PATTERNS**: Reuse components and layouts across pages
7. **PERFORMANCE FIRST**: Optimize for speed (lazy loading, caching)
8. **ACCESSIBLE**: WCAG 2.1 AA compliance for all users
9. **MOBILE-READY**: Responsive design for tablet/mobile CSMs
10. **PREMIUM FEEL**: Modern aesthetics that justify enterprise pricing

Framework: React 18 with TypeScript

Icons: Heroicons or Lucide React
Date Handling: date-fns or Day.js
Build Tool: Vite (for fast development)
